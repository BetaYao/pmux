# amux-ios — Thin iOS Client Design Spec

## Overview

A thin iOS client for AMUX that lets users monitor and interact with agents running on their laptop from their phone. All agents run on the macOS desktop app; iOS is a remote viewer and command sender.

**Key constraints:**
- iOS 26 target, pure SwiftUI, Liquid Glass design system
- Independent repo (not monorepo with macOS app)
- Cloud MQTT broker for communication (works on any network)
- Text-only terminal display (no ANSI rendering)
- Prompt-only input (no raw terminal input)

## Architecture

### Repository & Project

- **Separate repository:** `amux-ios`
- **Pure SwiftUI**, iOS 26+, no UIKit
- **Standard Xcode project** (no XcodeGen)
- **Dependencies:** MQTT client library (e.g. CocoaMQTT or MQTTNIO)
- Two-way MQTT protocol doc aligns iOS and macOS data formats

### MQTT Communication

**Cloud broker** (e.g. HiveMQ Cloud, EMQX Cloud) — TLS encrypted, token auth.

**Topic structure:**

```
# Desktop → iOS (subscribe)
amux/{deviceId}/worktrees                          # worktree list + status (QoS 1, retain)
amux/{deviceId}/worktree/{name}/terminal           # viewport text per pane (QoS 0)
amux/{deviceId}/worktree/{name}/status             # agent status changes (QoS 1)
amux/{deviceId}/worktree/{name}/members            # collaborator list (QoS 1, retain)

# iOS → Desktop (publish)
amux/{deviceId}/command/{worktree}/{paneId}/prompt  # send prompt to agent (QoS 1)
amux/{deviceId}/command/{worktree}/{paneId}/cancel  # abort current task (QoS 1)
amux/{deviceId}/command/{worktree}/settings         # update worktree settings (QoS 1)
amux/{deviceId}/command/worktree                    # create/delete worktree (QoS 1)
```

**Key decisions:**
- `deviceId`: UUID generated by macOS app, stored in config. iOS binds via QR code or manual input. One iOS app can bind multiple laptops.
- Terminal text follows macOS StatusPublisher cadence (~2s), no extra polling.
- `worktrees` topic uses retain so iOS gets latest list on connect.
- Commands bound to pane level (prompt, cancel) to target specific agent instances.

**Payloads (JSON):**

Worktree list:
```json
{
  "worktrees": [{
    "name": "feat-auth",
    "branch": "feat/auth-flow",
    "status": "agent_active",
    "lastPrompt": "implement login",
    "lastMessage": "Building auth module...",
    "paneCount": 2,
    "duration": "12m",
    "todoProgress": null
  }]
}
```

Terminal viewport:
```json
{
  "worktree": "feat-auth",
  "paneId": "pane-0",
  "paneIndex": 0,
  "totalPanes": 2,
  "lines": ["$ claude", "> Building auth module...", "✓ Created AuthService.swift"],
  "timestamp": 1706000000
}
```

## Screens

### 1. Main Screen — Worktree List

iOS 26 system style: light `#f2f2f7` background, SF Pro typography.

**Navigation bar:**
- Large title: "amux" (34pt bold)
- Right side: two glass circle buttons — Todo/Ideas (grid icon) and + New Worktree (plus icon)

**Device selector:**
- Horizontal scrollable glass chips below nav
- Green dot = online, gray = offline
- Tap to switch between bound laptops

**Worktree cards:**
- Large cards (~220pt), Instagram-style, scrollable list
- Glass material: `rgba(255,255,255,0.65)` + blur, `0.5px` border, `20pt` corner radius

Card layout (fixed height, top to bottom):
1. **Repo name** (left, 20pt semibold) + **Status badge** (right top, colored capsule)
2. **Last prompt** — user's most recent prompt, single line truncated
3. **Last message / todo** — agent output or todo progress "☑ 3/5 · current task", up to 2 lines
4. **Branch** (left, git icon + name) + **Panes** (right, dot visualization + count) — separated by top border line

Status colors (iOS system semantic colors):
- Active: `#248a3d` (systemGreen)
- Waiting: `#a05a00` (systemYellow)
- Idle: `#8e8e93` (tertiaryLabel)
- Error: `#d70015` (systemRed)

Tap card → push to worktree detail.

### 2. Worktree Detail — Terminal View

Same light `#f2f2f7` background, iOS 26 glass chrome.

**Navigation bar:**
- Left: glass circle back button (chevron, `#007aff`)
- Center: worktree name + status tag (e.g. "feat-auth `Active`")
- Right: glass circle + button (add pane)

**Pane indicator:**
- Centered dots (like Photos swipe indicator)
- Active dot: `#007aff`, inactive: `#c7c7cc`
- Swipe left/right to switch between panes

**Terminal panel:**
- Dark card inset (`#1c1c1e` background, 16pt corner radius) filling available space
- Monospace font (SF Mono, 12px), syntax-colored text lines
- Scrollable, displays plain text lines from MQTT viewport data

**Bottom toolbar — two groups:**

Left: glass capsule group (single container, icon-only, dividers between):
- ⚙ Settings icon — opens worktree settings sheet
- 👥 Member icon — opens collaborator list
- ✕ Abort icon (red `#ff3b30`) — cancels current agent task

Right: glass circle input button (pencil icon, `#007aff`)

**Input expanded state** (tap input button):
- Bottom area becomes glass expanded input panel
- Header: "Send to pane 1" label + ✕ close button
- Multi-line text area
- Footer: `@ file` and `# context` shortcut pills + blue send circle button
- Sends prompt to `amux/{deviceId}/command/{worktree}/{paneId}/prompt`

### 3. Todo & Ideas — Sheet

Presented as a sheet from main screen (left nav button).

**Navigation:**
- Glass circle close button (✕) top left
- Title: "Todo & Ideas"
- Segmented control: Todo | Ideas (iOS glass segmented style)

**Todo tab:**
- Grouped list sections: Today / This Week / Backlog
- Glass grouped list cards (system grouped style)
- Each row: priority dot (red/orange/green) + checkbox + title + worktree tag + assignee avatar (right) + date
- Completed items: checked, strikethrough, gray text
- Glass FAB bottom-right for adding new todo

**Ideas tab:**
- Card-based layout (not list rows)
- Each card: title + description (up to 3 lines) + tags + date
- Glass FAB for adding new idea

**Data storage:**
- Todo and Ideas stored locally on iOS (Core Data or SwiftData)
- Assignee field for future collaboration support
- Not synced via MQTT in v1

## macOS Side Changes

The macOS AMUX app needs a new **MQTT Publisher module**:

1. **MQTTPublisher** service — connects to cloud broker, publishes worktree list, terminal text, and status changes
2. **MQTTCommandHandler** — subscribes to command topics, routes prompts to correct pane, handles cancel/settings/worktree commands
3. **Config additions:** broker URL, credentials, deviceId (auto-generated UUID)
4. Hook into existing `StatusPublisher` pipeline — when viewport text is read for status detection, also publish to MQTT

## Design System Reference

All UI elements follow iOS 26 Liquid Glass:
- Glass material: `rgba(255,255,255,0.65)`, `backdrop-filter: blur(20px)`, `border: 0.5px solid rgba(0,0,0,0.06)`
- Inner highlight: `box-shadow: 0 0.5px 0 rgba(255,255,255,0.8) inset`
- Corner radius: 20pt (cards), 16pt (terminal inset, list groups), 50% (circle buttons)
- System blue: `#007aff`
- Background: `#f2f2f7`
- Typography: SF Pro Display (titles), SF Pro Text (body), SF Mono (terminal)
- In SwiftUI: `.glassEffect()` modifier for glass materials

## Out of Scope (v1)

- ANSI terminal rendering / color codes
- Raw terminal input (only prompt text)
- MQTT sync for Todo/Ideas
- Push notifications for status changes
- Home screen widgets
- Voice input
- Prompt templates
